# lucky draw

## CSRF-TOKEN

> 所有的接口，在请求头中必须加上CSRF-TOKEN。
>
> 如何获取？
>
> 按公钥对当前时间戳进行RSA256，公钥文件：ftx-site/config/rsa_public_key.pem
> 后端会解析出时间戳，并判断当前时间 与 请求中携带的时间是否在一个合理范围内，如果是，则认为这是一个正常的请求
>
> 注意：`公钥文件` 与 `加密逻辑`，前端应该使用 不可逆混淆 来防止代码泄露
